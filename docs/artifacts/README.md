# Compiler Artifacts

Snapshot of intermediate files produced by the compilation pipeline. These
satisfy the deliverable: *"Test programs executables, source code, lex result
and grammar result files, asm file"*.

## Lex / parser output

| File | Stage | Generated by |
|------|-------|--------------|
| `lex.yy.cpp` | Flex output — DFA-driven scanner | `flex -o build/lex.yy.cpp src/lexer.l` |
| `parser.tab.cpp` | Bison output — LALR(1) parser tables and semantic actions | `bison -d -o build/parser.tab.cpp src/parser.y` |
| `parser.tab.h` | Token enum shared with the lexer | same as above |

## Per-program intermediate representations

For each test program, two artifacts are included:

| Program | LLVM IR | x86_64 / ARM64 assembly |
|---------|---------|--------------------------|
| `factor.pas` | `factor.ll` | `factor.s` |
| `fibonacci.pas` | `fibonacci.ll` | `fibonacci.s` |
| `isprime.pas` | `isprime.ll` | `isprime.s` |
| `pidigits.pas` | `pidigits.ll` | `pidigits.s` |

### How they were produced

```bash
# Build the compiler
make

# Produce LLVM IR (.ll) — done implicitly by the driver
./minipc tests/factor.pas -o factor

# Lower IR to native assembly (.s)
clang -S -O0 factor.ll -o factor.s
```

The IR is the output of our LLVM C++ codegen pass (`src/codegen.cpp`); the
assembly is what `clang` emits when lowering the IR to the host architecture
(macOS arm64 in this snapshot).

## Reproducing

```bash
make clean
make
./build_and_test.sh
clang -S *.ll
```
